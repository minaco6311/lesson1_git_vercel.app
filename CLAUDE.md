# プロジェクトルール

## CHANGELOG の記録ルール

commit・push を行ったら、必ず `CHANGELOG.md` に変更内容を記録する。

### 記録のポリシー

- **新しい更新が上** になるよう記録する（降順）
- 後で見ても何をしたか、誰が見ても理解・再現できる粒度で書く
- 変更の「何を」「なぜ」「どのように」を要点としてまとめる

### CHANGELOG.md のフォーマット

```markdown
## [YYYY-MM-DD] タイトル（例: ログイン機能を追加）

### 変更内容
- 変更点を箇条書きで記載

### 背景・理由
- なぜこの変更を行ったか

### 再現手順（必要な場合）
1. 手順を記載
```

## Vercel Git運用ルール

Vercel連携後に `.gitignore` が未追跡になっている場合は、以下を確認してください。

* `.gitignore` の中に `.vercel` が書かれているか確認する
* 書かれていなければ `.vercel` を追加する
* `.vercel` フォルダ自体は Git に含めない
* `.gitignore` だけを `git add` する
* コミットメッセージは「.gitignoreにVercel設定を追加」にする
* その後、必要に応じて `git push` する

実行前に `git status` で状態を確認し、余計なファイルを一緒にコミットしないこと。
